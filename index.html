How Process Documentation Can Give You A Better Web Experience

Process documentation, standard operating procedures, hats write-ups and walkthroughs are all synonyms for a simple and often visual document that shows an individual how to get a job done correctly.

Before pivoting into web development, I was very much an operations guy. One of my first jobs was managing an offshore call center a dozen strong. I've since managed developers, copywriters, and whole ecommerce teams using processes and project management software.

If I were to sum up one thing I've learned, it would be that documentation can be invaluable when done right. That means it must be:

  • Written in simple language,
  • Key words must be defined at the outset,
  • Includes visual cues like screenshots or video, and
  • The process should be reviewed to establish certainty.
When it comes to assisting you, documentation can provide you with a very simple understanding of my coding strategies, lessons I've learned for an efficient process and effective tactics to best utilize what I've built.

In other words, you will save time, energy, and money.

  • I've used documentation at a NYC marketing agency to train interns how to update a WordPress website.
  • I've created documentation with clients showing them how to use software add-ons for their website.
  • I've written documention to help onboard future web developers as the team scales in size.
I can help you build a website and document my processes providing you with an easy and efficient workflow.
